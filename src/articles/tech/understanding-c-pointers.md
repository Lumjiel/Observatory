# 理解 C 语言指针

指针是 C 语言中最强大也是最容易被误解的特性之一。本文通过图解和代码示例，帮助你建立对指针的直观理解。

## 什么是指针？

指针是一个变量，其值为另一个变量的地址。听起来有点绕，让我们用图来说明。

```
普通变量:
┌─────────┐
│    42   │  name = "x", value = 42
└─────────┘

指针变量:
┌─────────┐
│ 0x1000  │  ptr = &x (存储的是 x 的地址)
└─────────┘
      │
      ▼
┌─────────┐
│    42   │  x 在内存中的位置是 0x1000
└─────────┘
```

## 数组名与指针的区别

虽然数组名在很多情况下表现得像指针，但它们有本质区别：

```c
int arr[5] = {1, 2, 3, 4, 5};
int *ptr = arr;  // arr 退化为指向首元素的指针

// 区别 1: sizeof
printf("%zu\n", sizeof(arr));   // 20 (5 * 4字节)
printf("%zu\n", sizeof(ptr));   // 8 (指针大小)

// 区别 2: &arr vs arr
printf("%p\n", arr);     // 首元素地址
printf("%p\n", &arr);    // 整个数组地址（类型 int(*)[5]）
```

## 画内存图的重要性

当我画出了内存布局图之后，很多之前模糊的概念变得清晰了：

```
栈内存:
┌──────┬──────┬──────┬──────┬──────┐
│  arr[0] │  arr[1] │  arr[2] │  arr[3] │  arr[4] │
└──────┴──────┴──────┴──────┴──────┘
 0x100   0x104   0x108   0x10c   0x110

ptr 指向 arr[2]:
┌──────┐
│ 0x108 │  ptr 的值
└──────┘
```

## 常见的指针错误

1. **未初始化的指针**
   ```c
   int *p;  // 野指针，指向未知地址
   *p = 42; // 危险！可能崩溃
   ```

2. **空指针**
   ```c
   int *p = NULL;  // 安全做法
   if (p != NULL) {
       // 使用 p
   }
   ```

## 小结

- 指针存储的是地址
- 数组名在大多数情况下退化为指针，但不是指针
- `sizeof` 行为不同是区分它们的常用方法
- 画内存图是理解指针的有效方法
