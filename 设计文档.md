## 终端日志 · 实时观测站 完整开发设计文档

**版本**：v1.0  
**日期**：2026-05-04  
**设计目标**：打造一个以“学习过程实时监控”为核心隐喻的个人技术博客，通过终端日志的视觉风格与交互，诚实记录大二学生的日常学习轨迹，呈现专业、自律、持续成长的“不张扬的 nb 感”。

---

### 1. 产品定义与第一印象

- **定位**：公开的个人学习数据监控中心，而非传统博客。
- **一句话描述**：这是一个正在记录你学习活动的系统终端，所有文章、笔记、项目碎片都变成了可检索、可展开的日志条目。
- **第一印象目标**：访客打开网站即感知到“这里有一个人在系统性地管理自己的学习进程，而且持续了很久”。暗色终端界面、等宽字体、结构化日志条目共同营造一种冷静、专注、工程化的氛围。

### 2. 视觉设计系统

#### 2.1 色彩方案

| 角色 | 色值 | 用途 |
|------|------|------|
| 背景层 | `#0A0E14` | 主背景，终端底色 |
| 表面层 | `#111820` | 展开详情面板、卡片背景 |
| 边框/分割 | `#2A3340` | 分隔线、面板边框 |
| 主文字（终端绿） | `#00C896` | 日志正文、标题、正常状态 |
| 警告/错误（琥珀） | `#FFAA00` | 错误条目、进行中标记、未解决警示 |
| 里程碑/系统提示（品红） | `#FF55AA` | 里程碑、系统状态变更、特殊强调 |
| 注释/元数据（冷灰） | `#5C7A8A` | 时间戳、次要信息、标签说明 |

**亮色模式（羊皮纸终端）**：
- 背景 `#FDF6E3`，文字 `#2E6B4C`，表面 `#EEE8D5`，保持等宽终端风格，降低长时间阅读疲劳。

#### 2.2 字体系统

- **全局等宽**：`JetBrains Mono`、`Fira Code`、`SF Mono`，依次回退。禁用连字以保持终端原生感。
- **字号**：桌面基准 13px，行高 1.7；移动端基准 12px，代码块 11px。
- **粗细**：正文常规 400，标题/标签 600，时间戳 400。

#### 2.3 图标与装饰

- 不使用任何图形图标，所有符号均由 ASCII 字符或单色 emoji（仅限系统基础符号）替代。
- 状态标记：`✓`（完成）、`⚠`（进行中）、`★`（里程碑）、`✗`（中断）。

### 3. 页面架构与多视图系统

全站采用**单页应用（SPA）模式**，不同视图通过命令输入或导航切换，同一份日志数据以不同形态呈现。

#### 3.1 全局组件

**顶部状态栏**（固定）：
- 左侧：`[student@observatory] ● 在线`
- 右侧：`运行时间: 87天14小时 | 活跃进程: 3 | 最后检查: 2026-05-04 14:32:01`
- 移动端两行布局，字体缩小，信息精简。

**底部导航**（移动端固定）：
- 三个图标按钮：“日志”、“仪表盘”、“更多”，点击切换视图。

**命令行输入区**：
- 桌面端：页面顶部固定 `$ grep -i [输入框] _`。
- 移动端：收起为浮动按钮，点击弹出半屏命令面板。

#### 3.2 视图列表

| 视图 | 命令/入口 | 功能说明 |
|------|------------|----------|
| 日志流 | 默认首页 | 所有日志条目倒序排列，支持筛选与展开 |
| 仪表盘 | `status` | 学习统计、热力柱状图、标签云、最近错误 |
| 错误看板 | `errors` | 仅显示 `[ERROR]` 条目，可批量标记已解决 |
| 里程碑时间轴 | `milestones` | 仅显示 `[MILESTONE]` 条目，垂直时间轴 |
| 项目展板 | `projects` | 以表格展示 `[BUILD]` 类项目进度与关联日志 |
| 技能树 | `neofetch` | ASCII 艺术树展示技术栈，叶子可交互跳转 |
| 关于 | `about` | 系统信息风格的个人介绍，模仿 `neofetch` 输出 |
| 帮助 | `help` | 列出所有可用命令与键盘快捷键 |

### 4. 内容类型与元数据规范

所有内容统称为“日志条目”，每条日志包含以下字段：

```json
{
  "id": "log_2026_05_04_001",
  "timestamp": "2026-05-04T14:32:01+08:00",
  "type": "INFO",
  "typeLabel": "INFO",
  "description": "完成“图的遍历”作业 | 耗时2h | 错误: segfault x2",
  "tags": ["数据结构", "C"],
  "status": "done",
  "detail": "Markdown 格式的完整笔记",
  "related": ["log_2026_04_28_003"],
  "duration": "2h",
  "progress": null
}
```

**类型枚举**：`INFO`（学习笔记）、`READ`（读书进度）、`BUILD`（项目实验）、`ERROR`（错误复盘）、`MILESTONE`（里程碑）、`THINK`（思考碎片）、`SUMMARY`（自动生成的周/月总结）。

**状态枚举**：`done`（已完成）、`wip`（进行中）、`error`（未解决）、`dropped`（中断）。

### 5. 交互逻辑与命令系统

#### 5.1 完整命令集

| 命令 | 参数 | 功能 |
|------|------|------|
| `grep <关键词>` | 任意文本 | 全文搜索日志 |
| `filter <类型>` | INFO/BUILD/ERROR/… | 按类型过滤 |
| `status` | 无 | 跳转仪表盘 |
| `errors` | 无 | 跳转错误看板 |
| `milestones` | 无 | 跳转里程碑时间轴 |
| `projects` | 无 | 跳转项目展板 |
| `neofetch` | 无 | 跳转技能树 |
| `about` | 无 | 关于页 |
| `help` | 无 | 帮助页 |
| `clear` | 无 | 返回默认日志流 |
| `theme <dark|light>` | 主题名 | 切换亮/暗模式 |
| `export <format>` | txt/json | 导出当前过滤结果 |

交互细节：
- 输入 `/` 自动聚焦搜索框。
- `Tab` 自动补全命令或标签。
- 命令历史：`↑↓` 浏览历史记录，最多保留 20 条。

#### 5.2 日志条目交互

- **点击**：原地展开/折叠详情面板。同一时间只允许一个展开。
- **悬停标签**：显示该标签关联日志数量，点击快速过滤。
- **点击时间戳**：滚动定位并高亮该条目。
- **点击状态标记**：弹出该状态含义提示（如“⚠ 进行中”表示尚未闭环）。
- **详情面板内**：`Esc` 或点击 `[X]` 关闭；`j/k`、`↑↓` 在条目间移动焦点。

#### 5.3 键盘快捷键

| 按键 | 功能 |
|------|------|
| `j` / `↓` | 下一个日志条目 |
| `k` / `↑` | 上一个日志条目 |
| `Enter` | 展开/折叠当前焦点条目 |
| `Esc` | 关闭详情 / 取消焦点 / 清空筛选 |
| `Shift+J` / `Shift+K` | 快速滚动页面 |
| `/` | 聚焦搜索框 |
| `Ctrl+Enter` | 执行当前输入命令 |

#### 5.4 移动端手势

- **详情面板**：从屏幕底部滑入，占据 90% 高度，顶部有下拉手柄，向下拖拽关闭。
- **长按条目**：弹出操作菜单（复制链接、分享、在详情中打开）。
- **向右滑动详情面板左边缘**：关闭面板。

### 6. 数据源与自动化更新

#### 6.1 数据存储结构

```text
/data
  manual-logs.json       # 手动编写的里程碑、复杂思考
  auto-generated.json    # 构建时自动生成（Git commits、阅读进度等）
  projects.json          # 项目元数据与进度
  skills.json            # 技能树定义
```

#### 6.2 自动化管线

- **Git 提交驱动**：通过 GitHub Actions 在每次 push 时执行脚本，解析最近提交中以 `[LOG]` `[READ]` `[BUILD]` `[ERROR]` 开头的消息，自动追加到 `auto-generated.json`。
- **定时任务**：每周日 23:59 触发一个 Actions 工作流，统计本周数据并生成 `[SUMMARY]` 条目，追加到 `auto-generated.json`。
- **状态过期检测**：构建时检查所有 `⚠` 条目，若超过 14 天未更新，在条目上增加 `stale` 标记，并在仪表盘显示提醒。

#### 6.3 站点构建

- 使用 **Eleventy (11ty)** 或 **Astro**，从 JSON 数据文件生成静态 HTML。
- 页面内交互（筛选、展开、命令）由原生 JavaScript 在客户端完成，不依赖框架。
- JSON 数据直接内嵌在页面脚本中（小数据量），或通过 fetch 异步加载（数据量增长后）。

### 7. 移动端适配策略

#### 7.1 断点与布局

- 主断点：`768px`。以下为移动端布局，以上为桌面端。
- 日志条目：桌面端单列居中，最大宽度 800px；移动端全宽，左右 padding 8px。
- 状态栏：移动端分为两行，字体缩小至 11px。
- 命令输入区：移动端换为浮动按钮，点击弹出半屏面板，提供快捷命令按钮。

#### 7.2 交互适配

- 点击区域最小 44x44px（符合 iOS/Android 标准）。
- 拖拽关闭详情面板支持触摸手势，带有弹性过渡动画。
- 禁用移动端的双击缩放：`touch-action: manipulation`。
- 详情面板内代码块允许横向滚动，字体缩小至 11px。

#### 7.3 性能优化

- 首屏仅渲染最近 20 条日志，滚动到底部加载更多（虚拟滚动或简单追加）。
- 移动端禁用非关键动画（如脉冲效果），保留静态状态标记。
- 主题切换使用 CSS 类名切换，无延迟。

### 8. 技术栈与工程结构

```
terminal-observatory/
├── src/
│   ├── _data/            # 日志、项目、技能 JSON
│   ├── _includes/        # 公共模板片段
│   ├── layouts/          # 页面布局模板
│   ├── assets/
│   │   ├── css/          # 手写 CSS（变量、主题）
│   │   └── js/           # 客户端交互（命令、筛选、手势）
│   └── pages/            # 各视图入口（实际生成单页）
├── scripts/
│   ├── git-parser.mjs    # Git log 解析脚本
│   └── summary-gen.mjs   # 周报生成脚本
├── .github/workflows/
│   └── build.yml         # 自动构建与数据更新
├── netlify.toml / vercel.json
└── package.json
```

**核心依赖**：Eleventy（静态生成）、highlight.js + 自定义主题（代码高亮）、markdown-it（解析 detail 字段）。客户端不引入任何第三方库。

### 9. 辅助功能与彩蛋

- **系统健康度**：仪表盘内展示 `学习系统健康度：███░░ 56%`，由更新频率、错误解决率、进行中任务数加权计算。
- **终端音效反馈**：切换主题或执行命令时可选择播放极短的电子提示音（需用户手势激活 AudioContext）。
- **控制台彩蛋**：浏览器开发者工具 Console 输出 `“> 看到这里的人，祝你编译一次通过。”`
- **404 页面**：模拟 `command not found` 错误输出，附带返回首页的链接。
